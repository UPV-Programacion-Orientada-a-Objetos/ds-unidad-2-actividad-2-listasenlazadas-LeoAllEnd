DECODIFICADOR PRT-7

Sistema de Ciberseguridad Industrial para Decodificación de Protocolo de Telemetría

Versión 1.0

Resumen

El Decodificador PRT-7 es un sistema de software que implementa la decodificación de un protocolo industrial de telemetría oculta. El protocolo no transmite mensajes directamente, sino instrucciones para construirlos, utilizando un sistema de mapeo dinámico similar a las máquinas de cifrado clásicas.

Objetivo: Interceptar y decodificar tramas del protocolo PRT-7 transmitidas por un dispositivo Arduino, ensamblando el mensaje oculto mediante estructuras de datos implementadas manualmente.

Jerarquía de Clases (POO)

TramaBase (Clase Abstracta)

Propósito: Define la interfaz común para todas las tramas del protocolo.

Características clave:

Destructor Virtual Método procesar() virtual puro Polimorfismo 
class TramaBase {
public:
    virtual ~TramaBase() {}  // CRÍTICO para evitar memory leaks
    virtual void procesar(ListaDeCarga* carga, RotorDeMapeo* rotor) = 0;
    virtual const char* toString() const = 0;
};

TramaLoad (Clase Derivada)

Propósito: Representa una trama de carga (L,X) que contiene un fragmento de datos.

Responsabilidades:

  • Almacenar un carácter
  • Solicitar decodificación al RotorDeMapeo
  • Insertar el resultado en ListaDeCarga
class TramaLoad : public TramaBase {
private:
    char caracter;
public:
    void procesar(ListaDeCarga* carga, RotorDeMapeo* rotor) override {
        char decodificado = rotor->getMapeo(caracter);
        carga->insertarAlFinal(decodificado);
    }
};

TramaMap (Clase Derivada)

Propósito: Representa una trama de mapeo (M,N) que modifica el estado del rotor.

Responsabilidades:

  • Almacenar un valor de rotación (positivo o negativo)
  • Aplicar la rotación al RotorDeMapeo
class TramaMap : public TramaBase {
private:
    int rotacion;
public:
    void procesar(ListaDeCarga* carga, RotorDeMapeo* rotor) override {
        rotor->rotar(rotacion);  // Cambia el estado del sistema
    }
};

Estructuras de Datos Manuales

1. RotorDeMapeo - Lista Circular Doblemente Enlazada

    ┌───┐    ┌───┐    ┌───┐    ┌───┐    ┌───┐
    │ A │◄──►│ B │◄──►│ C │◄──►│...│◄──►│ Z │
    └─┬─┘    └───┘    └───┘    └───┘    └─┬─┘
      │                                     │
      └─────────────────────────────────────┘
        (Cierre del círculo)

    cabeza → Apunta al nodo 'cero' actual
✅ Ventaja clave: Al rotar, NO se mueven los datos, solo se mueve el puntero cabeza. Esto hace la operación O(1) en complejidad.

Operación de Rotación:

void RotorDeMapeo::rotar(int n) {
    n = n % tamano;           // Normalizar
    if (n < 0) n += tamano;   // Manejar negativos
    
    for (int i = 0; i < n; i++) {
        cabeza = cabeza->siguiente;  // Solo mover puntero
    }
}

Operación de Mapeo (Cifrado César Circular):

Paso Operación Descripción
1 Buscar carácter de entrada Encontrar el nodo que contiene el carácter
2 Calcular distancia Desde cabeza hasta el nodo encontrado
3 Aplicar misma distancia Desde cabeza en dirección forward
4 Retornar carácter El carácter en la posición calculada

2. ListaDeCarga - Lista Doblemente Enlazada

cabeza                                              cola
  │                                                  │
  ▼                                                  ▼
┌───┐    ┌───┐    ┌───┐    ┌───┐    ┌───┐    ┌───┐
│ H │◄──►│ E │◄──►│ L │◄──►│ L │◄──►│ O │◄──►│...│
└───┘    └───┘    └───┘    └───┘    └───┘    └───┘

Cada nodo tiene:
- dato (char)
- siguiente (puntero)
- previo (puntero)

Operación de Inserción al Final (O(1)):

void ListaDeCarga::insertarAlFinal(char dato) {
    NodoCarga* nuevo = new NodoCarga(dato);
    
    if (cola == nullptr) {
        cabeza = nuevo;
        cola = nuevo;
    } else {
        cola->siguiente = nuevo;
        nuevo->previo = cola;
        cola = nuevo;
    }
    tamano++;
}

Gestión de Memoria

Principios Aplicados

  • Propiedad Clara: Cada estructura es dueña de sus nodos
  • RAII: Destructores liberan memoria automáticamente
  • Sin Memory Leaks: Cada new tiene su delete

Destructor Virtual - ¿Por qué es CRÍTICO?

// Sin destructor virtual:
TramaBase* trama = new TramaLoad('H');
delete trama;  // ❌ Solo llama ~TramaBase(), MEMORY LEAK

// Con destructor virtual:
class TramaBase {
    virtual ~TramaBase() {}  // ✅ CORRECTO
};

TramaBase* trama = new TramaLoad('H');
delete trama;  // ✅ Llama ~TramaLoad() correctamente
Regla de Oro: Si una clase tiene al menos un método virtual, su destructor DEBE ser virtual.
